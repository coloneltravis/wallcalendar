Getting started

The wall calendar plugin provides a month-by-month view of several calendar items. The idea originates from the "Sasco" wall charts which are often seen in offices to show annual leave and other appointments for staff. It would also be well-suited to a room booking application to show the availability dates for rooms in a hotel.

The plugin is written in Javascript/jQuery so will run entirely on the client. It can easily be configured with a JSON-formatted data feed which can either be generated dynamically by a server-side script or a static text file. As the user navigates to different months, the plugin will make a request for a new set of event data, so it is not necessary to provide a single file for a whole year of data.

The UI elements of the plugin such as the navigation buttons, headings and days can all be modified by editing the CSS file.

First things first, you will need tell the web browser to load the jQuery, the plugin and its CSS by placing the following code in the <head> of the page.

	
		
Page Header

		<script type="text/javascript" src="../js/jquery.min.js"> </script>
		<link rel="stylesheet" media="all" type="text/css" href="css/wallcalendar.css" />
		<script type="text/javascript" src="js/wallcalendar.plugin.js"> </script>
	


	
		
Example 1 - Annual leave calendar


		$('#leavechart').wallcalendar({

			    eventsfeed:'leaveevents.cfm',

				rows: [
					  	{id:101,title:'David Cameron'},
					  	{id:102,title:'Nick Clegg'},
					  	{id:103,title:'George Osborne'},
					  	{id:104,title:'Vince Cable'}
					  ],

					legend: [
							{id:1,tooltip:'Full day',classname:'full',colour:'#f00'},
							{id:4,tooltip:'Flexi day',classname:'flexi',colour:'#ff0'},
							{id:3,classname:'public',tooltip:'Public holiday',colour:'#f0f'},
							{id:5,classname:'half',tooltip:'Half day',colour:'#0ff'}
						]
		});

The following example shows how the plugin can be initialised as a room bookings calendar. Instead of specifying a path to a file, the data for each room is passed in JSON format to the eventsfeed option. In reality, this would not be an ideal implementation since the Javascript would need to be edited whenever a room is booked. 

	
		
Example 2 - Room bookings calendar

			$('#roombookings').wallcalendar(){

					eventsfeed: [
						{id:"room1", start:"20140906", end:"20140908", type:"1"},
						{id:"room2", start:"20141005", end:"20141005", type:"1"},
						{id:"room3", start:"20141101", end:"20141107", type:"1"},
						{id:"room4", start:"20140720", end:"20140725", type:"1"}
					],

					rows:[
							{id:'room1',title:'Harlech'},
							{id:'room2',title:'Beaumaris'},
							{id:'room3',title:'Caernarfon'},
							{id:'room4',title:'Caerphilly'}
						],

					legend: [
								{id:1,tooltip:'Booked',classname:'roombooked',colour:'#f00'}
						]
			});
		});
